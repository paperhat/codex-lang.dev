# Codex Glossary (G-I)

## Terms

### General annotation

An annotation that is neither attached nor grouping; it stands alone.

---

### Gloss

The Gloss Inline Semantic Span-Binding Language, a Paperhat product for inline semantic span annotations.

---

### Governing schema

The explicit schema used to perform semantic validation and define meaning for a Codex document.

---

### Grouping annotation

A single-line annotation with canonical text `GROUP: <label>` or `END: <label>` that marks an editorial region.

---

### Host Name Value

A value spelling `host(...)` that represents an ASCII DNS hostname.

---

### id trait

The language-level trait that assigns an IRI identifier to an Entity.

---

### Identity

The ability to refer to a Concept via explicit identifiers (`id`) or document-scoped keys (`key`).

---

### IdentityConstraint

A constraint type that enforces identity rules.

---

### IdentityError

An error class for violations of identity rules, such as missing or duplicate identifiers.

---

### Initial

The compatibility class required for the first schema version in a lineage.

---

### Inline annotation

A single-line annotation using `[` and `]` on the same line.

---

### Instance graph

The RDF graph produced by mapping a Codex document to RDF as defined by the specification.

---

### IRI Reference Value

An unquoted token containing a `:` that represents an IRI reference and is treated as opaque identity.

---
