# Codex Glossary (M-O)

This file contains glossary entries for terms starting with M-O.

See `../index.md` for format notes and other ranges.

[A–C](../a-c/index.md) | [D–F](../d-f/index.md) | [G–I](../g-i/index.md) | [J–L](../j-l/index.md) | **M–O** | [P–R](../p-r/index.md) | [S–U](../s-u/index.md) | [V–Z](../v-z/index.md)

## Terms

### Map Value

A collection of key/value pairs spelled as `map[key: value, ...]`.

---

### Marker

A syntactic delimiter for Concept instances in the surface form: opening, closing, or self-closing.

---

### MemberCount

A collection constraint that constrains the number of members in a collection.

---

### Named Color Value

A Color Value spelled as `&name` using a keyword from the named color list.

---

### Namespace

A camelCase identifier that binds to an imported schema within a `SchemaImports` block. Used as a prefix in qualified names to reference Concepts and Traits from the imported schema. Each schema declares its canonical namespace via its `namespace` trait (§11.3).

---

### Node IRI

The deterministic skolem IRI assigned to a Concept instance node in the instance graph (nodeIri).

---

### Node shape IRI

The deterministic IRI used for a node shape derived from a concept class IRI.

---

### Non-breaking change

A schema change that preserves all validation and semantic guarantees required by its declared compatibility class.

---

### Not

A rule combinator that inverts the truth of its child rule.

---

### Numeric Value

A numeric literal value whose spelling is preserved exactly and not evaluated.

---

### Opening marker

A `<Concept ...>` marker that begins a block Concept.

---

### OrderConstraint

A constraint type that enforces ordering rules.

---

### Ordered collection ($Ordered)

A collection whose member order is semantically significant and must be preserved.

---
